# Drag-to-Group Conversations Feature — Implementation Plan

## Overview

Users can drag a conversation on top of another conversation (or onto an existing group) to create a named group. The group name is auto-generated by the task model. Groups appear **inline within time sections** (Today, This week, etc.), positioned by their most-recently-updated conversation. Available to all instances (not gated behind HuggingChat). Mobile uses long-press (500ms) to initiate drag.

## 2. API Design

### Groups CRUD

**`GET /api/v2/conversation-groups`** — List user's groups with embedded conversation data

- Returns `{ groups: ConvGroupSidebar[] }`
- Each group includes its conversations (sidebar projections), sorted by updatedAt desc
- Group's `updatedAt` = max of its conversations' updatedAt values

**`POST /api/v2/conversation-groups`** — Create a new group

- Body: `{ conversationIds: string[], name?: string }`
- Sets `groupId` on the specified conversations
- If no name provided, generates one via task model (async, returns immediately with "New Group" placeholder)
- Returns `{ group: ConvGroupSidebar }`

**`PATCH /api/v2/conversation-groups/[id]`** — Update group

- Body: `{ name?: string, isCollapsed?: boolean }`
- Returns updated group

**`DELETE /api/v2/conversation-groups/[id]`** — Delete group

- Unsets `groupId` on all conversations in the group
- Deletes the group document

### Group Membership

**`PATCH /api/v2/conversation-groups/[id]/conversations`** — Modify group membership

- Body: `{ add?: string[], remove?: string[] }`
- `add`: sets `groupId` on those conversations (also unsets from any previous group)
- `remove`: unsets `groupId` on those conversations
- If group becomes empty after remove, auto-deletes the group
- Returns updated group (or `{ deleted: true }` if auto-deleted)

### Group Name Generation

**`POST /api/v2/conversation-groups/[id]/generate-name`** — Generate name via task model

- Fetches conversation titles from the group
- Calls task model with prompt (see Section 5)
- Updates group name and returns it
- Returns `{ name: string }`

### Conversations Endpoint Change

**`GET /api/v2/conversations`** — Exclude grouped conversations

- Add filter: `groupId: { $exists: false }` to the query
- This ensures ungrouped conversations paginate normally and grouped ones come via the groups endpoint

## 3. UI/UX Interaction Design

### Drag Mechanics (Pointer Events, no external library)

We use Pointer Events (not HTML5 DnD) for cross-platform consistency and fine-grained control.

**Desktop flow:**

1. `pointerdown` on a NavConversationItem → record start position, set `pendingDrag = true`
2. `pointermove` → if distance > 5px from start, enter drag mode:
   - Call `setPointerCapture()` on the element
   - Create a floating ghost element (fixed position, follows pointer)
   - Original item becomes semi-transparent (opacity 0.4)
   - Add `cursor: grabbing` to body
3. While dragging, track pointer position and hit-test against potential drop targets:
   - Another conversation item → highlight with blue ring + "merge" icon
   - A group header → highlight with blue ring + "add to group" indicator
   - Empty space → no highlight (drop cancels)
4. `pointerup` → execute action based on current drop target:
   - On conversation: create new group with both conversations
   - On group header: add conversation to that group
   - On nothing: cancel, animate ghost back to original position

**Mobile flow:**

1. `pointerdown` → start 500ms timer
2. If pointer moves > 5px during timer → cancel (it's a scroll), allow normal scroll behavior
3. If 500ms elapses without movement → enter drag mode:
   - Trigger haptic feedback (`navigator.vibrate(50)` if available)
   - Same ghost element + visual feedback as desktop
   - `touch-action: none` applied to prevent scroll interference
4. Same drop logic as desktop

### Visual Feedback During Drag

**Ghost element** (`DragOverlay.svelte`):

- Fixed position div following pointer with 8px offset
- Shows conversation title with a subtle shadow and slight rotation (2deg)
- Semi-transparent background matching sidebar style
- z-index: 9999

**Drop target highlights:**

- Conversation item: `ring-2 ring-blue-400 bg-blue-50 dark:bg-blue-900/20` with a small merge icon
- Group header: `ring-2 ring-blue-400 bg-blue-50 dark:bg-blue-900/20` with a "+" badge
- Transition: 150ms ease for ring/bg changes
- Invalid targets (dragging onto self, onto same group member): no highlight

**Source item during drag:**

- `opacity-40 pointer-events-none`

### Group Creation Animation

When two conversations merge into a group:

1. Ghost element animates to the target position (200ms ease-out)
2. Both conversation items slide together with a brief scale pulse
3. Group container expands around them (height transition)
4. Group name shows loading shimmer "Naming..." then resolves to AI name

### Group Visual Design in Sidebar

```
▾ ┃ ML Research                 3    [✏️] [🗑️]
  ┃   Fine-tuning GPT               [✏️] [🗑️]
  ┃   LoRA adapters                  [✏️] [🗑️]
  ┃   RAG pipeline                   [✏️] [🗑️]
```

- **Group header**: chevron (▾/▸) + colored left border accent + group name + conversation count badge
- Height: same `h-[2.15rem]` as conversation items (max-sm: h-10)
- Left border: 2px solid blue-400 accent on the group container
- Background: subtle `bg-gray-50/50 dark:bg-gray-800/30` on the group block
- **Collapsed state**: shows only the header row with count badge
- **Expand/collapse**: click on the header row (not on edit/delete buttons)
- **Hover actions**: edit (rename) and delete (ungroup all) buttons, same pattern as NavConversationItem
- **Child conversations**: indented with `pl-4` additional padding
- Children are individually draggable (to reorder between groups or drag out)

### Ungrouping

- Drag a conversation out of a group and drop it on empty space → removes from group
- If last conversation is removed → group auto-deletes
- Context: group delete button ungroups ALL conversations (with confirmation)
- Individual conversation in a group can be right-click → "Remove from group" (stretch goal, not MVP)

### Coexistence with Time Sections

Groups are injected into the time section matching their `updatedAt` (= max of member conversations' updatedAt):

```
── Today ──
  Chat about dogs
  ▾ ┃ ML Research (3)         ← group's newest conv was updated today
    ┃   Fine-tuning GPT
    ┃   LoRA adapters
    ┃   RAG pipeline
  Recipe ideas

── This week ──
  ▸ ┃ Work Tasks (2)          ← group's newest conv was this week
  Travel plans
```

Sort order within a time section: by updatedAt, interleaved with ungrouped conversations.

### State Management

**Drag state store** (`src/lib/stores/dragState.ts`):

```typescript
interface DragState {
	isDragging: boolean;
	draggedConv: ConvSidebar | null;
	sourceGroupId: string | null;
	ghostPosition: { x: number; y: number };
	dropTarget: {
		type: "conversation" | "group";
		id: string;
	} | null;
}
```

The drag state is a writable Svelte store. NavMenu coordinates drag start/end. Each NavConversationItem and NavGroupItem registers as a potential drop zone by reporting its bounding rect to the drag coordinator.

## 5. Group Name Generation

### Prompt Design

**File: `src/lib/server/textGeneration/groupName.ts`** (new)

```typescript
preprompt: `You are a conversation group naming assistant.
Goal: Given a list of conversation titles, produce a very short name (2-4 words) that captures their shared theme or topic.

Rules:
- Output ONLY the group name text. No prefixes, labels, quotes, emojis, hashtags, or trailing punctuation.
- Use the language of the conversation titles.
- Write a concise noun phrase.
- Never include meta-words: Group, Folder, Collection, Category, Conversations.

Examples:
Titles: "Python string reversal", "List comprehensions", "Django ORM tips" -> Python Programming
Titles: "NYC weekend plan", "Best restaurants Manhattan", "Central Park guide" -> New York Travel
Titles: "Transformer工作原理", "BERT微调方法" -> Transformer与BERT

Return only the group name text.`

messages: [{ from: "user", content: `Conversation titles:\n${titles.map(t => `- "${t}"`).join("\n")}` }]
generateSettings: { max_tokens: 24, temperature: 0 }
```

### When to Generate

- **On group creation**: if no name provided, generate immediately
- **On explicit rename**: user can click "regenerate" or type manually
- **Not on membership change**: to avoid surprising name changes; user can manually trigger regeneration

### Fallback

- If task model fails: use first conversation's title + " & more"
- If `LLM_SUMMARIZATION !== "true"`: combine first two words of each title

## 8. Edge Cases

| Scenario                                     | Behavior                                                                                                 |
| -------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| Last conversation removed from group         | Group auto-deletes (server returns `{ deleted: true }`)                                                  |
| Conversation in group gets deleted           | Normal delete; if group becomes empty, auto-delete group                                                 |
| Drag conversation from group A to group B    | Remove from A (auto-delete if empty), add to B                                                           |
| Drag conversation onto itself                | No-op, cancel drag                                                                                       |
| Drag conversation onto group it's already in | No-op, cancel drag                                                                                       |
| Conversation created via "New Chat"          | Starts ungrouped; user drags to group later                                                              |
| Search/filter (future)                       | Groups flatten during search — show matching conversations inline                                        |
| Many groups (50+)                            | Groups are lightweight; all loaded in one query. Render performance is fine since most will be collapsed |
| Shared conversations                         | Groups are user-specific; sharing ignores group membership                                               |
| Anonymous users (sessionId only)             | Groups work via sessionId, same as conversations                                                         |
| Concurrent tabs                              | Group state refreshes on navigation via SvelteKit invalidation                                           |
